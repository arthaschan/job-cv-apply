# STAR故事银行

面试中可复用的真实故事，按能力维度分类。每次面试准备后追加新故事。

## 使用方法

1. 面试前：按JD要求从本库中选取匹配的故事
2. 面试中：用STAR结构讲述，突出数据和结果
3. 面试后：将验证过的新故事追加到对应分类

## 一、团队管理

### 故事1: 扭亏为盈——华东区650人团队

| 要素 | 内容 |
|------|------|
| **S** | 某大型汽车制造企业华东区，团队650人，业绩连续下滑，利润-15% |
| **T** | 3个月内止损，6个月内扭亏 |
| **A** | ①推行SOP标准化（覆盖门店运营、客户服务、库存管理）<br>②区域协调机制（每周区域经理例会，问题24h响应）<br>③激励机制改革（绩效与利润挂钩，淘汰末位） |
| **R** | 利润从-15%→+8%，团队流失率从35%→12% |
| **反思** | 关键是先稳人心再改流程——团队士气低的时候推SOP会被抵触 |

**适用JD关键词**：团队管理、扭亏为盈、P&L、区域运营、SOP推行

## 三、AI落地 / 变革管理

### 故事3: 推动SOP数字化——从业务痛点到AI解决方案

| 要素 | 内容 |
|------|------|
| **S** | 用户运营团队处理15000+客户反馈，全靠人工分类，效率低、错误率高 |
| **T** | 用AI技术提升反馈处理效率 |
| **A** | ①梳理业务流程，标注数据（2000+样本）<br>②与技术团队协作，训练分类模型<br>③SOP推行（培训+手册+区域协调） |
| **R** | 分类准确率从人工72%→模型89%，处理速度提升3倍 |
| **反思** | AI落地的关键不是技术多牛，是业务团队愿不愿意用——所以SOP推行比模型优化更重要 |

**适用JD关键词**：AI Adoption、Change Enablement、业务赋能、SOP、数字化转型

## 四、跨部门协调

### 故事4: 跨区域项目统筹——华东+华南联动

| 要素 | 内容 |
|------|------|
| **S** | 公司要求华东+华南两个区域统一执行新运营标准，但两个区域文化差异大，执行进度不一 |
| **T** | 3个月内完成标准化推行 |
| **A** | ①分别与两个区域管理团队沟通，了解阻力点<br>②制定分阶段推行计划（华东先行试点→华南跟进）<br>③建立跨区域沟通机制（周报+月度复盘会） |
| **R** | 按时完成，华东执行率95%，华南88%（高于公司预期） |
| **反思** | 跨区域协调的核心是尊重差异——不是强推一套方案，而是允许本地化调整 |

**适用JD关键词**：Cross-functional、Regional coordination、Stakeholder management、Change management

## 五、扭亏 / 危机处理

### 故事5: 门店危机——从关店边缘到区域标杆

| 要素 | 内容 |
|------|------|
| **S** | 华东区某核心门店连续3个月亏损，总部考虑关闭 |
| **T** | 2个月内止血，否则关店 |
| **A** | ①驻店一周，摸清问题（库存积压+人员冗余+服务差）<br>②砍掉30%SKU，优化库存<br>③调整排班，淘汰2名表现差的员工<br>④重新培训服务流程 |
| **R** | 第2个月扭亏，第3个月成为区域利润前三 |
| **反思** | 扭亏的第一步是止血，不是创新——先把最明显的问题解决掉 |

**适用JD关键词**：Turnaround、P&L、Operations、危机处理、门店管理
